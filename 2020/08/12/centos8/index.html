Centos8 服务器环境搭建

发表于 2020-08-12

安装mysql

安装Yum Repository：

wget https://repo.mysql.com//mysql80-community-release-el8-1.noarch.rpm

使用rpm来安装MySQL

rpm -ivh mysql80-community-release-el8-1.noarch.rpm

使用yum安装mysql服务

yum install mysql-server

检查是否已经设置为开机启动MySQL服务

systemctl list-unit-files|grep mysqld

设置开机启动

systemctl enable mysqld.service

启动服务

systemctl start mysqld.service

MySQL 安装过程中会为 root 用户生成一个临时密码，先找到mysqld.log文件

find -name mysqld.log

查看

sudo grep 'temporary password' /var/log/mysqld.log

进入mysql 修改密码

mysql -u root -p
ALTER USER 'root'@'localhost' IDENTIFIED BY 'your passowrd';

可以运行安全设置向导脚本，来修改安全选项

sudo mysql_secure_installation

创建远程账号设置密码

create user 'username'@'%' identified by 'your_password';

创建本地账号

create user 'username'@'localhost' identified by 'your_password';

查看mysql所有用户和用户权限

SELECT DISTINCT CONCAT('User: ''',user,'''@''',host,''';') AS query FROM mysql.user;

创建数据库

create database database_name;

赋予用户指定数据库的远程访问权限

grant all privileges on database_name.* to 'username'@'%';

删除远程用户

drop user 'user'@'%';

mysql8.0使用的身份验证插件为：caching_sha2_password，客户端工具使用的身份验证插件仍是mysql_native_password，解决：
修改mysql.user中的加密方式,　　　　　

alter user 'root'@'%' identified with mysql_native_password by 'root密码';

查看端口

show global variables like 'port';
/etc/my.cnf.d/mysql-server.cnf

增加

[mysqld]
port=7100

重启mysql

service mysqld restart

redis 安装

wget http://download.redis.io/releases/redis-5.0.7.tar.gz

解压

tar -xvf redis-5.0.5.tar.gz
cd redis-5.0.5/

编译测试

make test

有可能需要

yum -y install gcc automake autoconf libtool make
yum install gcc gcc-c++

报出信息，需要安装tcl

You need tcl 8.5 or newer in order to run the Redis test

sudo tar xzvf tcl8.6.1-src.tar.gz -C /usr/local/
cd /usr/local/tcl8.6.1/unix/
sudo ./configure
sudo make
sudo make install

重新回到

cd redis-5.0.5/

重新执行

make test

编译

make

安装，安装到指定目录

make PREFIX=/usr/local/redis install

复杂配置文件到安装目录

cp redis.conf /usr/local/redis

cd /usr/local/redis/

mkdir log

编辑redis.conf
以daemon方式独立运行

daemonize yes

注释掉bind 127.0.0.1可以使所有的ip访问redis
若是想指定多个ip访问，但并不是全部的ip访问，可以bind
redis3.2之后，redis增加了protected-mode，在这个模式下，即使注释掉了bind 127.0.0.1，再访问redisd时候还是报错

protected-mode no

密码

requirepass your_password

启动

./bin/redis-server ./redis.conf

查看redis启动是否成功

ps aux | grep redis

apache-tomcat-8.5.51.tar

java路径

which java

java 版本

java -version

离线下载jdk
安装

rpm -ivh jdk-8u91-linux-x64.rpm

检查是否安装成功

java -version

可以成功执行javac、java –version操作，不需要JDK环境变量的配置
否则 /etc/profile 末尾追加以下内容:

export JAVA_HOME=/usr/java/default
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

重新加载环境变量使配置生效

source /etc/profile

查看是否配置成功

echo $JAVA_HOME

/webapps 下面的文件都可以删除
修改 conf/logging.properties
handle 中需要删除的部分，注意后面的符号

3manager.org.apache.juli.AsyncFileHandler,
4host-manager.org.apache.juli.AsyncFileHandler,

handlers = 1catalina.org.apache.juli.AsyncFileHandler,
2localhost.org.apache.juli.AsyncFileHandler,
3manager.org.apache.juli.AsyncFileHandler,
4host-manager.org.apache.juli.AsyncFileHandler,java.util.logging.ConsoleHandler

Handler specific properties 中需要删除的部分
如下所示

3manager.org.apache.juli.AsyncFileHandler.level = FINE
3manager.org.apache.juli.AsyncFileHandler.directory = ${catalina.base}/logs
3manager.org.apache.juli.AsyncFileHandler.prefix = manager.

4host-manager.org.apache.juli.AsyncFileHandler.level = FINE
4host-manager.org.apache.juli.AsyncFileHandler.directory = ${catalina.base}/logs
4host-manager.org.apache.juli.AsyncFileHandler.prefix = host-manager.

Facility specific properties 中需要删除的部分
如下所示

org.apache.catalina.core.ContainerBase.[Catalina].[localhost].[/manager].level = INFO
org.apache.catalina.core.ContainerBase.[Catalina].[localhost].[/manager].handlers = 3manager.org.apache.juli.AsyncFileHandler

org.apache.catalina.core.ContainerBase.[Catalina].[localhost].[/host-manager].level = INFO
org.apache.catalina.core.ContainerBase.[Catalina].[localhost].[/host-manager].handlers = 4host-manager.org.apache.juli.AsyncFileHandler

安装nginx

wget http://nginx.org/download/nginx-1.17.6.tar.gz

安装必要插件

yum -y install gcc pcre pcre-devel zlib zlib-devel openssl openssl-devel

解压

tar -zxvf nginx-1.17.6.tar.gz

指定安装路径

./configure --prefix=/usr/local/nginx

编译

make

安装

make install

进入到安装nginx目录下面的sbin
启动命令

./nginx

重启：

./nginx –s reload

停止：

./nginx –s stop

测试配置文件是否正常：

./nginx –t

强制关闭：

$ pkill nginx
